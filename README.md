### **6. [Fusion Retrieval: Combining Vector Search and BM25 for Enhanced Document Retrieval](https://www.aionlinecourse.com/ai-projects/playground/fusion-retrieval-combining-vector-search-and-bm25-for-enhanced-document-retrieval)**

**Project Overview**  
This project enhances document retrieval by combining semantic search (FAISS) and keyword-based ranking (BM25) to improve accuracy and efficiency. It processes PDF documents using advanced text and image extraction techniques, including PyMuPDF and LangChain’s RecursiveCharacterTextSplitter. The system then utilizes Hugging Face’s MiniLM model to generate vector embeddings for semantic search stored in FAISS, while BM25 scoring ensures keyword-based relevance. Additionally, a Large Language Model (LLM) is employed to generate context-aware summaries, enhancing the retrieval process with DeepSeek-R1-Distill-Qwen-1.5B.

The fusion of FAISS-based semantic search and BM25 ranking creates a hybrid retrieval approach that ensures comprehensive and relevant results, ideal for academic research, legal analysis, enterprise knowledge retrieval, and AI-driven Q&A systems.

#### **Key Features**  
- **Text Extraction:** Extracts text, tables, and images from PDFs using PyMuPDF and pdfplumber.
- **Semantic Search:** Generates vector embeddings with Hugging Face’s MiniLM model and stores them in FAISS for fast similarity searches.
- **Keyword-Based Ranking:** BM25 ranking refines retrieval by matching exact keywords, ensuring document relevance.
- **Hybrid Retrieval:** Combines FAISS semantic search and BM25 ranking for enhanced search accuracy.
- **LLM-Powered Insights:** Generates context-aware summaries using DeepSeek-R1-Distill-Qwen-1.5B for improved search results.
- **Citations:** Provides retrieved documents with citations, ensuring traceability and research transparency.

#### **Project Outcome**  
The Fusion Retrieval system effectively combines semantic and keyword-based retrieval methods to enhance document accuracy and relevance. By integrating AI-powered insights, it optimizes document retrieval for various applications, providing efficient and contextually rich search results for academic and enterprise use.

For more details and a hands-on experience, visit the **[Fusion Retrieval Project](https://www.aionlinecourse.com/ai-projects/playground/fusion-retrieval-combining-vector-search-and-bm25-for-enhanced-document-retrieval)** on AI Projects.

- **Vector Embeddings:** OpenAI embeddings are used to convert text chunks into vector representations for semantic search.
- **Contextual Search:** A custom retrieval function fetches relevant chunks along with their surrounding context to improve retrieval completeness.
- **FAISS Vector Store:** The embeddings are stored in a FAISS vector store for fast and efficient retrieval.
- **Comparison:** The method is compared against traditional retrieval techniques to highlight improvements in context and coherence.

#### **Project Outcome**  
The system successfully enhances document retrieval by adding contextually overlapping windows around text chunks, improving the coherence and completeness of retrieved information. ### **12. [Corrective Retrieval-Augmented Generation (CRAG) with Dynamic Adjustments](https://www.aionlinecourse.com/ai-projects/playground/corrective-retrieval-augmented-generation-rag-with-dynamic-adjustments)**

#### **Project Overview**  
This project develops a Corrective Retrieval-Augmented Generation (CRAG) system to enhance query answering by integrating document retrieval, relevance evaluation, and web search. The system starts by loading and vectorizing a PDF document ("Image Recognition Using Deep Neural Network"), followed by a FAISS index for similarity-based retrieval. Using a custom relevance scoring mechanism driven by GPT-4o, the system adjusts its behavior based on the relevance of the retrieved document. If the relevance is low, it fetches refined knowledge from the web and combines both sources to generate a final answer with citations, ensuring comprehensive, contextually rich responses.

#### **Key Features**  
- **Dynamic Relevance Evaluation:** GPT-4o scores document relevance and decides whether to use the retrieved document, fetch web data, or combine both.
- **Hybrid Search:** Combines FAISS-based document retrieval with web search via DuckDuckGo to refine query results.
- **Final Answer Generation:** Uses a language model to generate answers, ensuring they are context-aware and backed by sources.
- **Citations Included:** All answers come with citations to maintain transparency and reliability.

#### **Project Outcome**  
The system generates high-quality, contextually enriched answers by dynamically adjusting its search and retrieval approach. By leveraging both document-based and web-based knowledge, the CRAG system ensures that users receive the most relevant, accurate, and well-cited answers to their queries. This method proves particularly useful for specialized topics like image recognition, where the depth and precision of the information required are high.
